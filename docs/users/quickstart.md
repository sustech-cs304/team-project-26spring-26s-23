# 5 分钟上手

- 这页给谁看：已经准备动手试一下 CanDue 的普通使用者。
- 这页解决什么问题：用最短路径带你完成第一次可工作的聊天，并告诉你怎样判断是否成功。
- 当前覆盖到哪：覆盖桌面宿主里的首次进入、模型设置入口、默认模型选择和第一次发送消息；不展开安装分发或本地开发启动细节。
- 当前状态：首次聊天主线已可用；安装分发与不同交付方式的获取路径部分接通。

如果你已经拿到可运行的桌面应用，第一次上手通常只需要做四件事：打开应用、配置一个可用模型、回到助手工作区、发送第一条消息。

## 开始前先确认两件事

### 你已经拿到一个可运行的桌面宿主

这页默认你已经能打开 CanDue 桌面界面。当前文档不把安装分发方式写成统一成品流程，因为不同场景下，你可能拿到的是：

- 已经打包好的桌面应用。
- 项目成员本地跑起来的开发版。

如果你手上只有仓库代码，还没有把桌面宿主跑起来，请改读[给开发者](../developers/getting-started.md)。

### 你准备好了至少一个可用模型服务

CanDue 当前不会默认替你预置完整模型链路。你需要自己准备：

- 一个可用的 Provider 服务。
- 对应的 API Key 或等价凭据。
- 至少一个你能调用的模型。

## 最短操作路径

### 第一步：打开应用，进入设置工作区

打开 CanDue 后，先进入“设置”。如果你刚进入应用就看到还没有可用模型，这属于当前正常情况，不代表出错。

### 第二步：在“模型服务”里新增一个可用配置

在“模型服务”分区里：

1. 新增一个模型服务配置。
2. 选择你要使用的 Provider 类型。
3. 填入地址、密钥和模型信息。
4. 至少保存一个可用模型。

如果你不确定 Provider、profile、route 之间是什么关系，先跳去看[模型与 Provider 设置](./model-setup.md)；如果你要看完整定义，再读[Provider 与模型路由说明](../reference/providers-and-routing.md)。

### 第三步：可选地设置默认模型

进入“默认模型”分区，把主助手模型设成你刚刚配置好的那条模型路由。这样回到聊天区时，更容易直接看到一个默认选择。

### 第四步：回到“助手”工作区

回到助手工作区后，按当前界面主线完成：

1. 选择一个智能体。
2. 新建或进入一条对话。
3. 在输入区确认当前模型不再是空白状态。
4. 输入一条简单消息，例如“请先介绍你当前能做什么”。
5. 发送。

## 跑通后你应该看到什么

下面这些现象通常说明第一次上手已经成功：

- 聊天输入区里能看到可选模型，而不是只有空白提示。状态：已可用。
- 发送后，回复会逐步出现，而不是一直卡在等待中。状态：已可用。
- 某些组合上，你还能看到 thinking 相关选择或可见推理片段。状态：部分接通。

## 如果没有跑通，先看哪种情况

| 你看到的现象 | 先怎么判断 |
| --- | --- |
| 模型选择区是空的 | 先回到[模型与 Provider 设置](./model-setup.md)，确认至少保存了一条可用模型。 |
| 发送后提示服务不可用 | 先确认桌面宿主已经正常运行，再让项目成员检查本地运行时。仓库本地启动方式见[给开发者](../developers/getting-started.md)。 |
| 能聊天，但看不到 thinking | 这不一定是错误。先看[聊天与 Thinking](./chat-and-thinking.md)，再看[Thinking 能力说明](../reference/thinking.md)。 |
| 设置里已经能保存，但聊天里还是不能选 | 先确认你保存的是可执行的模型配置，而不是只有目录项。完整口径见[Provider 与模型路由说明](../reference/providers-and-routing.md)。 |
