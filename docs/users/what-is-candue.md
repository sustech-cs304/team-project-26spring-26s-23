# CanDue 是什么

CanDue 是一个运行在桌面上的助手工作台。它能承载模型配置、聊天功能和一部分扩展入口，但它不是已经完全产品化的平台。

## 你可以把 CanDue 理解成什么

### 一个本地桌面形态的助手工作台

主形态是桌面应用，而不是纯网页服务。你进入的是一个带助手工作区、设置工作区和若干扩展工作区的本地应用界面。

### 一个围绕聊天功能组织的使用界面

最清楚的一条主线是：

1. 配置模型服务。
2. 选择可用模型。
3. 在助手工作区里发起聊天。
4. 在支持的模型与 Provider 组合上使用 thinking。

### 一个还在继续补齐外围能力的工作台

除了聊天和模型设置，界面里还能看到 Blackboard、TIS、外部源、文件、开发等方向的入口。但这些区域的成熟度并不一致，它们目前是部分接通的状态。

## CanDue 不是什么

### 一个已经完全产品化的平台

简单说：主工作台已可用，外围能力仍在继续接通。它还不是所有能力都已经稳定、统一、成体系交付的平台。

### 一个打开就全部配好的成品

首次进入可以是空白模型配置：

- 还没有 Provider 配置。
- 还没有默认模型路由。
- 聊天区也可能先提示你去完成设置。

### 一个所有模型都有同样 thinking 体验的系统

thinking 已经是能力的一部分，但是否支持、支持成什么样，取决于你选择的模型与 Provider 组合。不同组合的档位、预算和展示效果并不完全一致。需要统一事实时，请看[Thinking 能力说明](../reference/thinking.md)。

## 适合谁

CanDue 适合下面这类使用者：

- 想在本地桌面界面里使用智能助手。
- 愿意自己配置模型服务。
- 需要一个能同时承载聊天、设置和后续扩展入口的工作台。
- 能接受一部分外围能力还处在部分接通状态。

## 以下情况需要先确认

如果你期待的是下面这些体验，请注意它们目前的状况：

- 开箱即用、已经预置完整模型链路的成熟成品。
- 所有 Provider 与模型都提供统一的 thinking 体验。
- Blackboard、TIS、外部源等区域都已经形成稳定产品化能力。
- 跨重启、跨工作区都很成熟的完整历史与数据闭环。

## 你能直接期待什么

| 主题 | 说明 | 状态 |
| --- | --- | --- |
| 桌面工作台 | 已经有明确的桌面应用、助手工作区和设置工作区。 | 已可用 |
| 模型设置 | 已经可以维护 Provider 配置、模型列表和默认模型。 | 已可用 |
| 聊天功能 | 已经可以进入聊天并接收流式回复。 | 已可用 |
| thinking | 已经能在支持的组合上选择或观察 thinking 行为。 | 已可用 |
| 外围能力 | 已经有页面或入口，但成熟度不一致。 | 部分接通 |
