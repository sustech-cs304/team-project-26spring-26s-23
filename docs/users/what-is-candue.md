# CanDue 是什么

- 这页给谁看：第一次接触 CanDue、想先判断它是不是自己现在需要的普通使用者。
- 这页解决什么问题：说明 CanDue 当前是什么、适合什么场景，以及不该怎样理解。
- 当前覆盖到哪：覆盖桌面宿主上的助手工作台主线，不展开安装分发、系统架构或未来平台化方案。
- 当前状态：桌面助手工作台主线已可用；外围能力与产品化接入面部分接通。

CanDue 现在更适合被理解成一个运行在桌面宿主里的助手工作台。它已经能承载模型配置、聊天主线和一部分扩展入口，但还不适合被写成一个已经完全产品化、能力都已收口的平台。

## 你可以把 CanDue 理解成什么

### 一个本地桌面形态的助手工作台

当前主形态是桌面宿主，而不是纯网页服务。你进入的是一个带助手工作区、设置工作区和若干扩展工作区的本地应用界面。

### 一个围绕聊天主线组织的使用界面

当前最清楚的一条主线是：

1. 配置模型服务。
2. 选择可用模型。
3. 在助手工作区里发起聊天。
4. 在支持的模型与 Provider 组合上使用 thinking。

### 一个还在继续补齐外围能力的工作台

除了聊天和模型设置，界面里还能看到 Blackboard、TIS、外部源、文件、开发等方向的入口。但这些区域的成熟度并不一致，不能都按“已经完整接通”来理解。

## 不要把 CanDue 理解成什么

### 一个已经完全产品化的平台

当前更准确的说法是：主工作台已可用，外围能力仍在继续接通。它还不是那种所有能力都已经稳定、统一、成体系交付的平台。

### 一个打开就全部配好的成品

当前首次进入可以是空白模型配置：

- 还没有 Provider 配置。
- 还没有默认模型路由。
- 聊天区也可能先提示你去完成设置。

### 一个所有模型都有同样 thinking 体验的系统

thinking 已经是当前能力的一部分，但是否支持、支持成什么样，取决于你选择的模型与 Provider 组合。不同组合的档位、预算和展示效果并不完全一致。需要统一事实时，请看[Thinking 能力说明](../reference/thinking.md)。

## 当前最适合谁

CanDue 当前最适合下面这类使用者：

- 想在本地桌面界面里使用智能助手。
- 愿意自己配置模型服务。
- 需要一个能同时承载聊天、设置和后续扩展入口的工作台。
- 能接受一部分外围能力还处在部分接通状态。

## 当前不太适合谁

如果你期待的是下面这些体验，当前就需要保守判断：

- 开箱即用、已经预置完整模型链路的成熟成品。
- 所有 Provider 与模型都提供统一的 thinking 体验。
- Blackboard、TIS、外部源等区域都已经形成稳定产品化能力。
- 跨重启、跨工作区都很成熟的完整历史与数据闭环。

## 当前你能直接期待什么

| 主题 | 现在可以怎样理解 | 当前状态 |
| --- | --- | --- |
| 桌面工作台 | 已经有明确的桌面宿主、助手工作区和设置工作区。 | 已可用 |
| 模型设置 | 已经可以维护 Provider 配置、模型列表和默认模型。 | 已可用 |
| 聊天主线 | 已经可以进入聊天并接收流式回复。 | 已可用 |
| thinking | 已经能在支持的组合上选择或观察 thinking 行为。 | 已可用 |
| 外围能力 | 已经有页面或入口，但成熟度不一致。 | 部分接通 |
