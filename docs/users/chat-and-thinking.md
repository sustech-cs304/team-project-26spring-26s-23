# 聊天与 Thinking

聊天区已经是可以用的主线。你可以选择模型、发送消息、看到流式回复；如果模型与 Provider 组合支持，还会看到 Thinking 相关选择或可见推理片段。不同组合的体验并不完全一致。

## 日常聊天时，你通常会做什么

### 第一步：进入助手工作区

主线在"助手"工作区里。你会先选择一个智能体，再进入或新建一条对话。

### 第二步：确认模型不是空白状态

如果聊天输入区里还没有可选模型，先不要急着发消息，先回到[模型与 Provider 设置](./model-setup.md)补齐配置。

### 第三步：按需要调整 Thinking

如果模型与 Provider 组合支持 Thinking，界面里可能会出现：

- 开关式选择。
- 档位式选择。
- 预算式选择。

如果组合不支持，你也可能完全看不到相关控件。这属于正常现象。

### 第四步：发送消息并观察过程

发送后，你看到的是逐步推进的回复过程，而不是最后一次性吐出整段文本。你还可能看到：

- 文本逐步出现。
- 中间穿插工具步骤。
- 某些组合上出现可见 reasoning。

## 重启后历史会怎样恢复

聊天历史默认保存在本地数据里。对使用者来说，体验是：

- 重启应用或本地后端后，左侧历史线程列表会重新出现。
- 打开历史线程时，时间线会按需从本地记录恢复，而不是依赖这次运行仍然在线。
- 如果某条历史 run 已结束，你可以查看它对应的基础 replay / 历史快照信息。
- 如果历史线程使用的模型、工具或 Thinking 能力现在已经不可用，界面会单独提示 drift，而不是悄悄替换成当前配置。

**历史记录保存的是当时真实发生过的配置和事件，不是重新推断出来的"最接近结果"。**

## 怎样理解 Thinking

### Thinking 已经是能力的一部分

这不是旧文档里的历史概念，也不是只存在于计划稿里的能力。它已经进入主线。

### 但它不是所有模型都一样

准确的理解是：Thinking 是否支持、支持成什么形式、是否展示 reasoning，都取决于**模型与 Provider 的组合**。

所以你可能遇到下面这些都正常的情况：

- 这个模型能选 Thinking，另一个模型没有这个入口。
- 两个模型都有 Thinking，但一个是档位，另一个更像预算。
- 你已经启用了 Thinking，但界面没有展示推理过程。

如果模型/Provider 的 API 本身不输出推理内容，运行时不会展示。此时 run 元数据中的 `reasoningSuppressionBasis` 会说明原因。
完整定义请看[Thinking 能力说明](../reference/thinking.md)。

## 怎样判断看到的是正常现象

| 你看到的现象 | 准确的理解 |
| --- | --- |
| 回复是一段段出来的 | 这是流式聊天方式，属于正常现象。 |
| 先出现工具步骤，再继续回答 | 这代表回复里发生了工具调用，也是正常主线的一部分。 |
| 某个模型没有 Thinking 入口 | 这通常表示该模型与 Provider 组合不支持，或界面不展示这一能力。 |
| 开了 Thinking 但没看到推理过程 | 运行时已完整支持 `reasoning_delta` 事件流；如果未展示，通常是因为模型/Provider 的 API 未输出推理内容，可查看 run 元数据中的 `reasoningSuppressionBasis` 确认原因。 |
| 不同模型的 Thinking 选项不一样 | 这是实现的正常边界，不同模型的 Thinking 选项不是同一套统一档位。 |

## 工具步骤该怎么理解

当聊天里出现工具步骤时，你可以把它理解成"系统为了回答你，先调用了一个工具动作"。从使用者视角，最重要的是看三个结果：

- 它有没有开始执行。
- 它最后是完成还是失败。
- 最终回答有没有继续正常完成。

如果你关心底层事件名称和顺序，再去看[运行时接口 / 事件参考](../reference/runtime-events.md)。

## 需要注意的几点

### 推理过程展示取决于模型/Provider 的 API

运行时已完整支持通过 `reasoning_delta` 事件流展示推理过程。是否输出推理内容取决于模型/Provider 的 API，不展示时 run 元数据中的 `reasoningSuppressionBasis` 会给出原因。

### 不同模型的 Thinking 体验不是一套统一档位

主线已经接入 Thinking，但不同组合仍会表现出不同的选择形式和展示效果。

### 聊天旁路区域尚未全部成为成熟产品能力

聊天功能已可用，但外围的 Blackboard、TIS、外部源、文件、开发等区域尚未全部成熟。
