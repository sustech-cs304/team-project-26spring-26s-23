# 聊天与 Thinking

- 这页给谁看：已经配好模型、准备日常使用 CanDue 聊天能力的普通使用者。
- 这页解决什么问题：说明聊天时你会执行什么动作、看到什么结果，以及 Thinking 为什么在不同模型上表现不同。
- 当前覆盖到哪：覆盖助手工作区里的日常聊天流程、Thinking 的用户可见行为、工具步骤的可见结果；不展开底层事件协议。
- 当前状态：聊天主线、本地历史恢复与基础 replay 已可用；可见 reasoning 与跨组合一致体验部分接通。

先说结论：当前聊天区已经是可用主线。你可以选择模型、发送消息、看到流式回复；如果当前模型与 Provider 组合支持，还会看到 Thinking 相关选择或可见推理片段。但不同组合的体验并不完全一致。

## 日常聊天时，你通常会做什么

### 第一步：进入助手工作区

当前主线在“助手”工作区里。你会先选择一个智能体，再进入或新建一条对话。

### 第二步：确认当前模型不是空白状态

如果聊天输入区里还没有可选模型，先不要急着发消息，先回到[模型与 Provider 设置](./model-setup.md)补齐配置。

### 第三步：按需要调整 Thinking

如果当前模型与 Provider 组合支持 Thinking，界面里可能会出现：

- 开关式选择。
- 档位式选择。
- 预算式选择。

如果当前组合不支持，你也可能完全看不到相关控件。这在当前是正常现象。

### 第四步：发送消息并观察过程

发送后，你当前更可能看到的是一个逐步推进的回复过程，而不是最后一次性吐出整段文本。你还可能看到：

- 文本逐步出现。
- 中间穿插工具步骤。
- 某些组合上出现可见 reasoning。

## 重启后历史会怎样恢复

当前聊天历史已经默认保存在本地持久化数据里。对普通使用者来说，更直接的体验是：

- 重启应用或 hosted runtime 后，左侧历史线程列表会重新出现
- 打开历史线程时，时间线会按需从本地记录恢复，而不是依赖这次运行仍然在线
- 如果某条历史 run 已结束，你可以查看它对应的基础 replay / 历史快照信息
- 如果历史线程当时使用的模型、工具或 Thinking 能力现在已经不可用，界面会单独提示 drift，而不是悄悄替换成当前配置

这里最需要记住的一点是：**历史记录保存的是当时真实发生过的配置和事件，不是今天重新推断出来的“当前最接近结果”。**

## 现在应该怎样理解 Thinking

### Thinking 已经是当前能力的一部分

这不是旧文档里的历史概念，也不是只存在于计划稿里的能力。当前它已经进入主线。

### 但它不是所有模型都一样

当前更保守、也更准确的理解是：Thinking 是否支持、支持成什么形式、是否展示 reasoning，都取决于**模型与 Provider 的组合**。

所以你可能遇到下面这些都正常的情况：

- 这个模型能选 Thinking，另一个模型没有这个入口。
- 两个模型都有 Thinking，但一个是档位，另一个更像预算。
- 你已经启用了 Thinking，但界面没有展示完整 reasoning。

完整定义请看[Thinking 能力说明](../reference/thinking.md)。

## 怎样判断当前看到的是正常现象

| 你看到的现象 | 更准确的理解 |
| --- | --- |
| 回复是一段段出来的 | 这是当前流式聊天主线，属于正常现象。 |
| 先出现工具步骤，再继续回答 | 这代表当前回复里发生了工具调用，也是正常主线的一部分。 |
| 某个模型没有 Thinking 入口 | 这通常表示该模型与 Provider 组合当前不支持，或界面不展示这一能力。 |
| 开了 Thinking 但没看到完整 reasoning | 不一定是失败，也可能是当前组合只支持请求，不保证完整展示。 |
| 不同模型的 Thinking 选项不一样 | 这是当前实现的正常边界，不应该强行当成同一套统一档位。 |

## 当前工具步骤该怎么理解

当聊天里出现工具步骤时，你可以先把它理解成“系统为了回答你，先调用了一个工具动作”。从使用者视角，最重要的是看三个结果：

- 它有没有开始执行。
- 它最后是完成还是失败。
- 最终回答有没有继续正常收口。

如果你关心底层事件名称和顺序，再去看[运行时接口 / 事件参考](../reference/runtime-events.md)。

## 当前最值得保守判断的地方

### 不要把“支持 Thinking 请求”和“稳定展示完整 reasoning”写成同一件事

前者更像能力支持，后者还要看当前组合和展示策略。二者当前不能简单画等号。

### 不要把不同模型的 Thinking 体验强行看成一套统一档位

当前主线已经接入 Thinking，但不同组合仍会表现出不同的选择形式和展示效果。

### 不要把所有聊天旁路区域都写成成熟产品能力

聊天主线已可用，不代表外围的 Blackboard、TIS、外部源、文件、开发等区域都已经成熟。
