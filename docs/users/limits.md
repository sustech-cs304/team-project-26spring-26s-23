# 当前限制与能力边界

如果你主要看重"桌面工作台 + 模型配置 + 助手聊天"这条主线，CanDue 已经值得直接上手。如果你期待所有外围能力都已经成熟接通，请注意它们目前是部分接通或规划中的状态。

## 已可用

| 主题 | 说明 |
| --- | --- |
| 桌面上的助手工作台 | 主形态已经成立，能进入助手与设置工作区。 |
| 模型服务配置 | 可以新增自己的 Provider 配置、保存密钥和模型列表。 |
| 默认模型设置 | 可以为聊天区提供默认模型路由。 |
| 聊天功能 | 可以选择模型、发送消息、接收流式回复。 |
| Thinking 基础能力 | 在支持的模型与 Provider 组合上，可以选择或观察 Thinking 行为。 |
| 本地聊天历史恢复 | 重启应用或后端后，线程列表、历史时间线和基础 run replay 可以从本地已保存数据恢复。 |

## 部分接通

| 主题 | 现状说明 |
| --- | --- |
| Thinking 的统一体验 | 不同模型与 Provider 组合的档位、预算和展示效果并不完全一致。 |
| 可见 reasoning 展示 | 运行时已完整支持通过 `reasoning_delta` 事件流展示；是否输出推理内容取决于模型/Provider 的 API。 |
| 历史线程继续对话 | 如果历史模型、工具或 Thinking 能力与当前配置发生漂移，需要先显式重新绑定后再继续发送。 |
| 备份与恢复 | 聊天历史支持本地 SQLite 备份 / 恢复，但它是单机运维能力，不是云同步。 |
| Blackboard 与 TIS 相关区域 | 已经有真实能力、设置或入口，但尚未形成完整成熟的产品面。 |
| 外部源与 WakeUP 等流程 | 已经有页面和局部交互，但尚未形成稳定闭环。 |
| 能力、文件、开发等工作区 | 页面结构已经有了，但真实数据面和业务闭环仍不完整。 |

## 规划中

| 主题 | 说明 |
| --- | --- |
| 跨设备 / 云端同步的完整历史恢复 | 本地保存不是多设备同步或云端历史。 |
| 围绕 Blackboard、TIS、外部源的完整产品化体验 | 现在尚未整体落地。 |
| 所有工作区都形成稳定成熟的数据面 | 尚未达到这个标准。 |

### 目录里有入口，但完整能力可能尚未成熟

你在界面里看到某个工作区、某个设置分区、某个领域入口，并不意味着它已经完成端到端闭环。

### 能配置，但不同组合结果可能不同

模型能配置、也能聊天，但不同模型与 Provider 组合可能给出不同的 Thinking、reasoning 或工具体验。

### 能保存字段，但下游可能尚未稳定生效

一些设置已经能稳定保存，但它们可能尚未成为成熟运行时功能。

### 本地历史恢复，不是云同步

恢复能力依赖本地数据库。它能帮你跨重启找回历史线程，但这些历史不会自动同步到其他设备。

## 如果你只想快速判断"现在适不适合我"

### 适合立即试用的情况

- 你想要的是桌面上的智能助手工作台。
- 你愿意自己配置模型服务。
- 你主要使用聊天功能，而不是期待外围能力全部成熟。

### 以下情况需要先确认

- 你期待所有外围工作区都已经和后端深度接通。
- 你期待 Blackboard、TIS、外部源都已经形成完整稳定产品面。
- 你期待所有模型都提供统一的 Thinking 与 reasoning 体验。
