# 赶渡 CanDue - 核心产品 UI 交互设计文档

## 概述体系 (Design System Guidelines)
- **色彩策略**: 迎合学生工具属性，主色调使用 SUtech 绿/深蓝 (Tech Blue)，辅色用中性灰。为满足“状态透明化”，AI 工作状态应用多种提示色：琥珀色（工具调用中）、绿色（动作成功）、红色（高危写操作阻断确认）。
- **布局规范**: 考虑到前端构建于 Electron 桌面上，采用桌面端最契合的 `全局侧边栏 (Global Layout) + 二级导航/任务池 (Secondary List) + 主业务区 (Detail Space)` 的响应式分栏结构。

---

## 1. 核心界面一：Agentic 智能交互工作台 
*(对应需求：智能体交互、思维轨迹展示、高风险用户确认)*

这是用户日常最高频停留的对话界面，通过独立的确认组件与思维链可视化折叠面板，来保证 AI 行为的可解释性和安全性。

### 1.1 UI 线框图 (Low-fidelity Wireframe)

```text
+---------------------------------------------------------------------------+
| ⚙️区 | 智能体 / 会话列表     | Blackboard 日程调度管家               [清空] |
|------|---------------------|--------------------------------------------|
| 💬   | [BB日程管家] ▾      |  User: 帮我规划一下下周三之前的物理作业。  |
| 助手 |   - 下周复习规划    |                                            |
|      |   - 物理实验协调    |  +--------------------------------------+  |
| 📅   |                     |  | ⚙️ 正在调用工具: fetch_bb_deadlines..|  |
| 日历 | [校园政策问答] ▾    |  |   ✓ 已获取 3 个截止日期                |  |
|      |   - 培养方案查询    |  | ⚙️ 正在调用工具: check_calendar_conflict| |
| 📁   |                     |  +--------------------------------------+  |
| 资料 | [复习资料分析] ▾    |  Agent: 下周三前您有物理实验。为您生成了排 |
|      |   ( + 新建会话 )    |  期草案，是否将复习块写入您的个人日历中？  |
| 🔧   |                     |                                            |
| 开发 |                     |  +--------------------------------------+  |
|      |                     |  | ⚠️ 安全确认：写入本地系统日历        |  |
| ⚙️   |                     |  | 动作: 添加3个事件到 "Study" 日历。   |  |
| 设置 |                     |  | [ 检查详情 ]   [ 拒绝 ] [ 授权执行 ] |  |
|      |                     |  +--------------------------------------+  |
|      |                     |                                            |
|      |                     | ------------------------------------------ |
|      |                     | [ 🔧 选择工具 ▾ ] [ 模型：GPT-4o ▾ ]       |
|      |                     | > 输入您的需求，或按 / 呼出控制指令...   ⏎ |
+---------------------------------------------------------------------------+
```

### 1.2 核心交互说明
- **轨迹透明化 (Traceability)**: 当 Agent 调用后端或 Blackboard API 时，渲染为一个带有旋转 Loading 图标的卡片。调用结束后自动折叠为一条小绿勾，极大地提升用户信任度。
- **高危操作阻断 (Safety Confirmation)**: 当识别到“写入日历”或“删除缓存”等动作时，中断 `run` 生命周期，呈现实体红框警告拦截卡，只有当用户点击“授权执行”时才放行。

## 2. 核心界面二：可视化日程审阅与协同排期 
*(对应需求：多源日程规划、可视化日程审阅与协同编辑)*

单纯从文字去了解 Agent 排了什么行程往往不够直观。此界面提供基于 AI 的可视化白板日历视图供用户人工微调。

### 2.1 UI 线框图 (Low-fidelity Wireframe)

```text
+---------------------------------------------------------------------------+
| 导航 | 任务池区 / 数据源   | <  2026年 4月  >          [返回对话] [保存] |
|------|---------------------|--------------------------------------------|
| 助手 | 数据源显示控制：    |          | 周一 (13) | 周二 (14) | 周三 (15) |
| 日历 | [☑] 个人定程 (蓝)   |  09:00   |----------|----------|-----------|
| 资料 | [☑] 学校课表 (紫)   |          | [个]组会 |          | [课]软工  |
|      | [☑] AI草案排期(绿)  |  11:00   |----------|----------|-----------|
| 设置 |                     |          |          |✨[AI建议]|           |
|      | 未排期 Backlog池：  |  13:00   |----------|  做物理  |-----------|
|      | :: 软工报告文档     |          |          |  实验    |           |
|      | :: 阅读论文         |  15:00   |----------|----------|-----------|
|      | (支持拖拽到右侧)    |          |          |          | [BB]大物  |
|      |                     |  17:00   |----------|----------|  作业Due  |
|      |                     |--------------------------------------------|
|      |                     |  ✨ AI 巡检提示: 拖拽成功，当前调整无时间冲突。 |
+---------------------------------------------------------------------------+
```

### 2.2 核心交互说明
- **多源数据视图融合**: 通过不同色块（如紫色固定课程，蓝色个人日常）区分。AI 自动生成的学习排期（图中的“✨AI建议”）使用明显的绿色发光高亮框，让 AI 的干预一目了然。
- **人在回回路协同 (Human-in-the-loop Editing)**: 用户对 AI 的排期不满意，可以直接鼠标按下拖拽该事件块换位置。每当发生拖拽，底部的 AI 巡检提示条会自动执行后台的冲突检测工具，若与上课时间重合会标红告警。

## 3. 核心界面三：沉浸式资料分析与问答区
*(对应需求：学习资料辅助分析、提取摘要及复习题)*

这是针对教育场景的工具，当学生阅读厚重的文献或 PDF PPT 时，能有一个“贴靠式”的分屏 AI 辅助面。

### 3.1 UI 线框图 (Low-fidelity Wireframe)

```text
+---------------------------------------------------------------------------+
| 导航 | 本地资料库列表      |  [PDF] 第四章：Agentic 架构    | 💡 资料分析助手     |
|------|---------------------|------------------------------|------------------|
| 助手 | 📚 当前课程资料     | 4.1 什么是 Agentic Loop      | AI: 我已阅读完本文.|
| 日历 | - 软工第四章.pdf    | 代理循环是一种基于自主思考...| 为您生成了3道重点测|
| 资料 | - 培养方案.pdf      | {此处是被拉蓝选中的文字...}  | 验卡片供复习：     |
|      | - 高数复习.md       |                              |                  |
| 开发 |                     | 4.2 常见工具调用模式         | +--------------+ |
|      | [+ 上传新资料并提取]| 包含检索、执行等操作。...    | | 什么是代理   | |
| 设置 |                     |                              | | 循环？       | |
|      |                     |                              | |   [点击翻转] | |
|      |                     |                              | +--------------+ |
|      |                     |                              |                  |
|      |                     |                              | [全文总结] [测验]|
|      |                     |                              | > 帮我解释这部分 ⏎ |
+---------------------------------------------------------------------------+
```

### 3.2 核心交互说明
- **上下文伴随 (Context-aware Reading)**: 左侧为文档渲染器区，右侧为 AI。当用户在左侧选本文段落时，右侧 AI 输入框会自动带入选中内容的引用，进行精细化答疑。
- **Flashcard 问答互动交互**: 针对“生成测验”功能，右侧不直接输出死板文字，而是渲染成带有“点击翻转”功能的互动式复习卡片 (Flashcard)。卡片翻转后不仅显示答案，还会附带相关跳转链接，满足“提供结果依据”的原始需求。
