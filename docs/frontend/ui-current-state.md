# 前端当前 UI 状态说明

这篇文档的目标，是帮助读者先建立一个清晰画面：**今天打开应用，真正会看到什么界面，它们分别承担什么角色。**

如果你想先理解字段存储位置，建议再配合字段参考文档一起看；如果你想先知道“当前前端到底长什么样”，这一篇最适合作为入口。

## 先记住四件事

当前前端 UI 的整体认识，可以先从下面四点开始：

1. **启动壳已经带有主题决策，而且窗口会在首屏准备完成后再进入显示阶段。**
2. **聊天入口已经稳定成 session-first 三栏壳。**
3. **设置页已经扩展成更大的 settings workspace，并形成一组稳定分区。**
4. **最近一轮收敛之后，一些旧设置项已经退出当前界面，同时也出现了新的正式入口。**

有了这个总印象，再去看各页细节会更容易理解。

## 启动阶段现在会看到什么

### 启动壳会先完成主题准备

当前启动过程可以按下面的顺序理解：

1. 先根据系统主题给启动壳一个浅色或深色兜底外观。
2. 再尝试读取公开配置中心中的正式主题。
3. 先渲染启动屏，再继续装配 React 主工作台。
4. 启动壳完成首屏绘制后，窗口再进入可显示状态。

这让启动体验更接近一个完整桌面应用：

- 冷启动可以直接以当前主题进入，而不是先固定成单一浅色外观。
- 深色主题已经进入启动体验本身。
- 首屏会在内容准备好后再展示给用户。

### 启动时当前可见的几类状态

当前用户在启动阶段最常见的可见状态包括：

- **准备桌面界面**：renderer 根装配层正在启动。
- **连接本地服务**：宿主正在准备本地后端。
- **可解释失败态**：如果根装配失败，会显示错误卡片和重载动作。

## 工作台总布局现在怎样理解

当前工作台仍然保持“左侧图标栏 + 右侧主内容区”的总布局。

左侧主入口包括：

- 助手
- 能力
- 文件
- 开发
- 设置

其中当前最接近真实业务主路径的，仍然是：

- **助手工作区**
- **设置工作区**

`capabilities`、`files`、`developer` 目前更适合作为结构化工作区来理解，它们已经有入口和布局，但当前重点仍然在信息架构承载上。

## 设置工作区现在到底长什么样

设置页是这一轮变化最明显的部分之一。现在更适合把它看成一个完整的 settings workspace，而不是几张分散卡片。

### 左侧导航现在有哪些分区

当前设置工作区左侧导航包括 12 个分区：

1. SUSTech 信息
2. 模型服务
3. 默认模型
4. 常规设置
5. 显示设置
6. 数据设置
7. MCP 服务器
8. 网络搜索
9. 全局记忆
10. API 服务器
11. 文档处理
12. 外部源

这组导航本身已经能帮助读者建立一个重要认识：

- 设置工作区现在覆盖范围更大。
- `SUSTech 信息` 和 `外部源` 都已经是当前真实页面。

### 各分区当前分别承担什么角色

| 分区 | 当前界面里能看到什么 | 当前更适合怎样理解 |
| --- | --- | --- |
| SUSTech 信息 | 学号、邮箱、CAS 密码、Blackboard 自动下载与大小限制、TIS 信息卡片 | 已有正式页面；其中 TIS 卡片仍在等待后续扩展 |
| 模型服务 | 服务商列表、搜索、添加、右键菜单、复制、删除、拖动排序、服务商详情、API 密钥输入、模型列表管理与模型编辑弹层 | 当前设置页里最完整的一块之一 |
| 默认模型 | 主助手模型、快速执行模型两个路由下拉框 | 当前聚焦默认模型路由 |
| 常规设置 | 界面语言、代理模式、助手消息通知、自动备份 | 当前页面已经收敛成基础常规偏好区 |
| 显示设置 | 主题切换 | 当前显示页聚焦主题 |
| 数据设置 | 数据目录、备份周期、自动备份、启动时同步 | 已有稳定页面形态 |
| MCP 服务器 | 工具权限策略、自动发现 MCP 服务 | 结构稳定，承载 MCP 相关工作区设置 |
| 网络搜索 | 默认搜索引擎、结果数量、压缩方式 | 结构稳定 |
| 全局记忆 | 记忆策略、自动清理陈旧记忆 | 结构稳定 |
| API 服务器 | 开发态 runtimeUrl 入口、根层启动摘要、重试按钮，以及本页自己的地址 / 重连 / 轮询字段 | 当前最典型的“公开配置 + 工作区状态”混合页 |
| 文档处理 | 导出格式、输出目录、自动生成文件名 | 已有稳定页面形态 |
| 外部源 | WakeUP 分享链接、解析按钮、解析结果弹窗 | 已有真实入口，后续会继续承载外部来源接入 |

### 模型服务页已经形成怎样的交互体验

模型服务页现在已经具备一套完整而清晰的工作区交互。

当前服务商列表支持：

- 搜索服务商、地址或模型
- 添加服务商
- 切换当前服务商
- 右键菜单
- 复制服务商
- 删除服务商
- 拖动排序
- 无服务商时显示空状态

服务商详情区支持：

- 编辑服务商基础信息
- API 密钥输入
- API 密钥显示 / 隐藏
- API 密钥复制
- 模型列表管理
- 模型新增、编辑、删除
- 模型编辑弹层中的基础字段与高级字段

其中一个很值得单独记住的事实是：

- **provider secrets 会回填到详情表单里。**

也就是说，模型服务页已经同时具备普通字段与 secret 字段的实际读写体验。

### `SUSTech 信息` 页可以先怎样理解

`SUSTech 信息` 当前已经形成三组明确内容：

1. **基本信息**：学号、邮箱。
2. **校园凭据**：CAS 密码。
3. **Blackboard 偏好**：自动下载、文件大小限制。

同页中的 `TIS 信息` 目前仍是等待扩展的子区，因此阅读时可以把这页理解成：

- 页面主体已经成立。
- 内部仍保留一块后续扩展位。

### `外部源` 页当前是什么状态

`外部源` 页面已经不只是导航中的一个标题。当前页面里已经可以看到：

- WakeUP 分享链接输入框
- 解析按钮
- 解析结果弹窗入口
- 解析失败提示
- 冲突处理按钮组

因此这页更适合作为“已有真实入口、后续继续扩展来源接入逻辑”的页面来理解。

## 设置页里最近完成了哪些收敛

这一轮设置页最明显的变化，是很多页面开始更聚焦于自己的主要职责。

### 常规设置现在聚焦基础偏好

当前常规页主要承载：

- 界面语言
- 代理模式
- 助手消息通知
- 自动备份

阅读当前常规页时，可以把重点放在基础偏好与工作区使用习惯上。

### 显示设置现在聚焦主题

当前显示页的界面内容收敛得很明确：

- 主题是当前页面中的可见主入口

因此这一页更适合作为“主题设置页”来理解。动画偏好仍然保留在系统层配置里，但当前显示页已经把视觉入口集中到主题本身；字号与界面密度也不在这页展示。

### 默认模型页现在聚焦模型路由

当前默认模型页中的核心内容是：

- 主助手模型
- 快速执行模型

阅读这一页时，可以把重点放在模型路由选择上；这样也更容易和宿主公开配置中的后端默认模型字段区分开。

## 助手工作区现在已经是什么样

### 当前整体结构是 session-first 三栏

当前助手工作区最适合用下面这张简单示意图来理解：

```text
┌────────────┬────────────┬──────────────────────┐
│ 智能体目录 │ 会话列     │ 聊天主内容区         │
└────────────┴────────────┴──────────────────────┘
```

### 左侧：智能体目录列

左侧当前显示的是：

- 后端返回的智能体目录
- 当前可选智能体
- 当前选中状态

这里的阅读重点是：目录真源已经在后端，前端负责的是展示与交互组织。

### 中间：会话列

中间列当前已经承担一组真实动作：

- 为当前智能体创建会话
- 展示当前窗口内的会话列表
- 切换当前激活会话
- 拖动排序会话
- 右键打开会话菜单

会话菜单中也已经有比较完整的结构，包括：

- 重命名会话
- 删除会话
- 生成会话名
- 复制会话子菜单
- 导出会话子菜单

这让会话列更像一个已经进入真实主路径的工作区组件。当前仍保留的边界是：会话列表主要存在于当前窗口内存态中。

### 右侧：聊天主内容区

右侧聊天区会根据当前连接状态、目录状态和会话状态，逐步进入不同阶段。它已经承担起当前聊天主路径中最核心的一段交互体验。

## 聊天面板当前会显示哪些阶段

### 根层尚在装配时

当前用户会看到几类很清晰的状态壳层：

- 等待根层装配
- 无可用 runtime URL
- 连接信息不完整
- 宿主正在启动后端
- 宿主启动失败
- 降级但仍可继续

这些状态说明已经围绕当前真正重要的条件展开：

- runtime URL
- hosted backend 状态
- 根层 retry

### 已进入连接态后

聊天壳会继续经历下面这些阶段：

1. 等待后端智能体目录
2. 目录失败时显示错误
3. 无可选智能体时显示占位
4. 无会话时提示先在左侧新建会话
5. 有会话后进入真正的消息发送壳

## 当前聊天发送区到底长什么样

### 1. 消息区已经是独立滚动区域

当前聊天区已经包括：

- 可滚动消息流
- 空会话提示
- 用户消息
- 助手消息
- 发送失败消息

### 2. 输入区已经是底部停靠式布局

当前输入区是一套完整的底部 composer，包含：

- 顶部工具栏
- 中间可调高度的输入区
- 发送按钮

并且当前支持：

- **Enter 发送**
- **Ctrl + Enter 换行**
- **拖动调整输入区高度**

### 3. 模型选择器已经进入消息发送主路径

模型选择器位于输入区工具栏中，当前已经具备：

- 搜索
- 标签筛选
- 分组展示
- 请求级模型选择

阅读时可以把它理解成“本次消息发送使用什么模型”的界面入口。

### 4. 工具选择器也已经进入消息发送主路径

工具选择器同样位于输入区工具栏中，当前支持：

- 搜索工具
- 全选
- 反选
- 推荐工具集
- 基于当前会话能力面的工具目录展示

它已经成为消息发送体验中的正式组成部分。
