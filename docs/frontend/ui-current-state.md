# 前端当前 UI 状态说明

这篇文档不讨论未来设计草案，只回答当前用户真正能看到什么、能点什么、哪些动作会进入正式链路。

如果你想先知道“页面长什么样”，读这篇最直接；如果你想先知道“字段归谁管”，再配合字段参考文档一起看。

## 先给结论

当前 UI 最值得先记住的四件事是：

1. 启动页已经支持按系统主题和配置中心主题进入浅色 / 深色外观。
2. 助手工作区已经进入 session-first 三栏形态，不再是旧全局 agent 入口。
3. 聊天面板已经有当前稳定的消息区、底部输入区、模型选择器和工具选择器布局。
4. 设置页里已经有少量正式入口，但大量表单仍然只是页面交互。

## 应用启动阶段现在长什么样

### 启动页

应用入口启动时，会先显示统一的启动壳。

当前启动页的顺序大致是：

1. 先根据系统主题给一个浅色或深色兜底外观。
2. 再尝试从配置中心读取正式主题。
3. React 根装配层开始渲染工作台。

这意味着当前夜间模式已经不再是“工作台渲染完再切”，而是冷启动阶段就已经进入外观决策。

### 启动时当前会看到哪些状态

- **准备桌面界面**：前端正在装配 renderer 根界面。
- **连接本地服务**：宿主正在启动本地后端。
- **根级错误卡片**：如果读取链路或工作台懒加载失败，会显示可解释失败态，而不是白屏。

## 工作台总布局

当前工作台仍然是“左侧图标栏 + 右侧主内容区”的结构。

### 左侧图标栏当前有哪些入口

主入口包括：

- 助手
- 能力
- 文件
- 开发
- 设置

其中当前成熟度最高的还是：

- 助手
- 设置

## 主题与显示相关 UI 现状

### 主题

主题现在已经不是临时本地状态，而是正式配置字段。

当前行为包括：

- 启动时会先应用兜底主题
- 成功读到配置中心后再切到正式主题
- 设置页中可切换浅色 / 深色
- 变更后立即影响工作台
- 保存失败时会回滚

### 动画开关

当前微动画开关也已经进入正式配置链路。

当前行为包括：

- 设置页中可开启 / 关闭
- 会立即影响根节点动画数据属性
- 保存失败会回滚

### 当前仍不是正式显示配置的内容

这些项虽然已经有 UI，但当前还主要是页面交互：

- 字号
- 界面密度

所以显示设置现在更准确的说法是：

- **主题和动画开关已经正式接入**
- **字号和密度还没有正式进入持久化闭环**

## 助手工作区现在长什么样

### 当前整体结构

当前助手工作区已经稳定成三栏：

```text
┌────────────┬────────────┬──────────────────────┐
│ 智能体目录 │ 会话列     │ 聊天主内容区         │
└────────────┴────────────┴──────────────────────┘
```

### 左侧：智能体目录列

当前这里显示的是：

- 后端返回的智能体目录
- 每个智能体的图标和名称
- 当前选中状态

要特别注意：

- 这里已经不能再写成“前端静态智能体列表”。
- 前端只是对后端目录项做展示增强。

### 中间：会话列

当前中间列已经不再是单纯占位。

它现在负责：

- 为当前选中的智能体创建会话
- 展示当前窗口内已有的会话列表
- 切换当前激活会话

但它当前还没有做到：

- 跨刷新恢复
- 从后端拉历史会话目录

所以中间列现在最准确的说法是：

- 已经进入真实交互主路径
- 但仍然是当前窗口内存态会话管理

### 右侧：聊天主内容区

右侧聊天区现在是整个助手工作区最成熟的部分。

它会先根据 bootstrap 状态显示不同壳层，再在条件满足后继续进入消息发送界面。

## 聊天面板当前状态 UI

### 1. `loading`

会显示：

- 正在等待根装配层完成运行态装配

### 2. `empty`

会显示：

- 还没有可用运行时
- 当前既没有宿主给出的地址，也没有开发态 override

### 3. `incomplete`

会显示：

- 当前连接信息仍不完整

当前更应该把它理解成：

- 还没拿到继续进入主路径所需的关键连接条件，主要还是 runtime URL

而不是旧说法里的“缺少全局 agentName”。

### 4. `starting`

会显示：

- 宿主正在启动本地后端
- 当前 hosted 状态摘要

### 5. `failed`

会显示：

- 宿主启动失败说明
- 最小失败摘要
- 根层统一 retry 动作

### 6. `degraded`

会显示：

- 宿主已经降级，但当前仍然保留可用地址
- 聊天主路径仍可继续

### 7. `ready`

当前不会再直接显示旧式“全局 agent 已就绪，可以直接开聊”的界面。

现在 `ready` 后会继续进入：

1. 智能体目录加载
2. 会话创建
3. 消息发送壳

## session-first 聊天壳当前 UI 长什么样

### 目录未加载完成时

聊天区会显示：

- 正在准备智能体目录

### 目录加载失败时

聊天区会显示：

- 后端智能体目录加载失败
- 不会回落到本地静态智能体列表

### 当前没有可选智能体时

聊天区会显示：

- 后端目录中暂无可选智能体

### 当前还没有创建会话时

聊天区会显示一个会话占位壳：

- 提示用户在左侧选择智能体并新建会话

### 当前已经有会话时

会进入真正的消息发送壳，包含：

- 可滚动消息流
- 空会话提示
- 底部输入区
- 发送按钮
- 模型选择器
- 工具选择器

## 聊天面板布局现在有哪些变化

### 1. 消息区

当前消息区已经是独立滚动区域，用于显示：

- 用户消息
- 助手消息
- 错误消息

### 2. 输入区

当前输入区位于底部，带有：

- 文本输入框
- Enter 发送
- Ctrl + Enter 换行
- 发送按钮

### 3. 模型选择器

模型选择器当前位于输入区工具栏中。

它的现状是：

- 当前是前端维护的请求级模型选择 UI
- 带搜索、标签过滤和分组显示
- 当前选择结果会进入 `message/send`

### 4. 工具选择器

工具选择器也位于输入区工具栏中。

它的现状是：

- 读取当前会话能力面中的工具目录
- 支持搜索、全选、反选和推荐工具集
- 当前选择结果会进入 `message/send` 的 `enabledTools`

### 5. 当前输入区不是哪些东西

不要把当前输入区写成：

- 旧 Provider 的包裹壳
- 只做样式展示的假输入框
- 不带真实模型 / 工具请求语义的装饰界面

## 设置工作区现在长什么样

### 左侧导航

当前设置工作区左侧包括：

- 模型服务
- 默认模型
- 常规设置
- 显示设置
- 数据设置
- MCP 服务器
- 网络搜索
- 全局记忆
- API 服务器
- 文档处理

### 当前已经正式接入的设置 UI

#### 显示设置

当前正式接入：

- 主题
- 微动画开关

#### 常规设置

当前正式接入：

- `agentName` 卡片

#### 默认模型

当前正式接入：

- 后端模型卡片，对应 `backendExposed.model`

#### API 服务器

当前正式接入：

- 开发态 `runtimeUrl` 卡片
- 根装配层状态摘要
- 根层 retry 动作

### 当前仍主要是本地交互的设置 UI

以下部分当前还不应写成已经进入正式配置闭环：

- 模型服务商编辑器
- 默认模型页中的主助手模型 / 快速执行模型下拉框
- 常规设置里的语言、代理、通知、自动备份
- 显示设置里的字号与密度
- 数据设置
- MCP 服务器设置
- 搜索设置
- 记忆设置
- API 页面中的后端地址、重连策略、健康检查轮询
- 文档处理设置

## 其余三个工作区现在是什么状态

### `capabilities`

当前主要是结构化占位页，用于表达未来能力工作区的大致布局。

### `files`

当前主要是结构化占位页，用于表达未来文件工作区的大致布局。

### `developer`

当前主要是结构化占位页，用于表达未来开发工作台的大致布局。

它们当前共同特点是：

- 有入口
- 有页面结构
- 但没有真实数据回流

## 当前不要再写成这些说法

下面这些写法现在都不准确：

- “启动阶段主要在等旧 Provider 注入。”
- “助手工作区左侧仍然是前端静态智能体列表。”
- “中间列还是纯占位，当前不会创建真实会话。”
- “聊天面板只是状态说明卡片，尚未进入消息发送界面。”
- “显示设置里只有主题已经正式接入。”
