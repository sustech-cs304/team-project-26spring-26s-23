# 页面能力参考

这页是速查表。它不重复解释页面长什么样，也不展开字段 schema，只用来回答“这一页今天做到哪一步”。界面结构请看[前端当前 UI 状态](./ui-current-state.md)，字段 owner 请看[当前生效字段参考](./reference-current-fields.md)。

## 先看三种页面状态

当前 frontend 页面更适合按下面三种状态来理解：

| 状态 | 含义 |
| --- | --- |
| 主路径页面 | 页面已经直接参与当前聊天、主题或连接链路。 |
| 正式持久化页面 | 页面已经有稳定入口，字段会保存，但下游消费范围仍然有限或不均衡。 |
| 半接通 / 占位页面 | 页面已经有入口或交互，但后端数据面、业务闭环或跨页面联动还不完整。 |

## 工作区总表

| 工作区 | 当前主要数据来源 | 当前状态 | 说明 |
| --- | --- | --- | --- |
| `assistant` | hosted runtime、后端智能体目录、会话能力面、当前窗口内会话状态 | 主路径页面 | 这是当前最完整的一条业务主线。 |
| `settings` | 公开配置中心 + settings workspace state + settings workspace secrets | 主路径页面和正式持久化页面并存 | 设置工作区已经稳定，但不同分区成熟度差异很大。 |
| `capabilities` | 前端静态内容 | 半接通 / 占位页面 | 已有结构化 Hub 页面，还没有真实能力数据面。 |
| `files` | 前端静态内容 | 半接通 / 占位页面 | 已有工作区骨架，还没有真实文件数据来源。 |
| `developer` | 前端静态内容 | 半接通 / 占位页面 | 已有开发工作台骨架，还没有真实任务、构建或日志数据面。 |

## 助手工作区的当前定位

`assistant` 已经可以明确写成当前前端主路径，因为它已经连成了下面这条链路：

1. 根装配层确认当前有可用 `runtimeUrl`。
2. 助手工作区拉取后端智能体目录。
3. 用户选择智能体并创建 `thread`。
4. 前端读取当前 `thread` 的能力面。
5. 聊天面板通过 `run/start` 发起本轮 run，并显式发送本次模型路由、Thinking 和工具选择。
6. 前端通过 `run/stream` 消费事件流；需要时再走 `run/cancel`。

`session/create`、`capabilities/get` 和 `message/send` 继续保留，但它们当前只是兼容壳。

当前仍然保留的边界主要有两条：

- 会话列表首先服务当前窗口内的会话操作。
- 历史消息回放、服务端会话目录和跨刷新恢复还没有成为成熟产品能力。

## 为什么 `模型服务` 和 `默认模型` 要单独提高优先级

这两页虽然也属于设置工作区，但它们和聊天发送区的关系已经比其他分区更直接。

`模型服务` 当前负责维护 provider profiles 与模型清单。聊天面板会把这些 provider profiles 展开成模型目录，因此它已经不是单纯的本地表单页。

`默认模型` 当前负责保存默认模型路由。聊天发送区会优先读取 `primaryAssistantModel`，再结合 provider profiles 找到当前首选模型。因此这页也已经进入聊天主路径。

同时也要记住它们的边界：首次状态仍然可以完全空白。没有 provider profiles 时，聊天区不会自动出现示例模型。

## 为什么 `API 服务器` 是混合页

`API 服务器` 当前同时包含两类内容：

- 公开配置中心里的开发态运行时覆盖地址。
- settings workspace 里的地址草稿、重连策略和轮询字段。

所以这页的一部分会直接影响连接判断，另一部分仍然主要是长期设置项。阅读时最好不要把整页都当成“已经全面接入 runtime 的配置页”。

## 为什么 `外部源` 仍然放在边界区

`外部源` 已经不是空白入口，因为用户确实可以输入 WakeUP 分享链接并触发解析弹窗。

但这条流程当前仍然主要停留在前端半接通阶段：

- 分享链接会保存到 settings workspace。
- 解析反馈主要由前端侧流程控制。
- 文档不能把它写成已经接入成熟后端同步能力的稳定页面。

## 其余三个工作区的当前定位

`capabilities`、`files` 和 `developer` 三个工作区都已经拥有统一的 Hub 页面结构。它们目前最重要的价值，是把未来会承载的主题和栏目位置先稳定下来。

因此当前更准确的写法是：

- 页面结构已经存在。
- 信息架构已经成形。
- 真实数据面和业务闭环仍未完成。
