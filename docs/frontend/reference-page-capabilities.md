# 页面能力参考

## 文档用途

这份附录用于快速回答：**五个工作区当前各自的数据从哪里来、交互做到什么程度、有没有正式持久化、哪些已经接入统一配置中心、哪些还是前端占位。**

它适合在这些场景使用：

- 新同学快速判断各页面成熟度
- 评审时核对“这页到底是事实、占位，还是未来方向”
- 规划开发顺序时确认哪些页面更接近主路径

## 使用边界

- 本文只描述当前页面能力，不补写不存在的数据源或接口。
- “是否持久化”按当前正式链路判断，不按页面外观判断。
- “是否接后端”按当前真实连接行为判断，不按字段名称猜测。
- 如果你想查字段或运行态，请分别看 [当前生效字段参考](./reference-current-fields.md) 和 [前端运行时状态参考](./reference-runtime-states.md)。

| --- | --- | --- | --- | --- | --- | --- |
| `assistant` | 左侧助手类型和话题列表来自前端静态常量；右侧聊天区消费根装配层的 bootstrap 状态 | 中等：可切助手、切话题、看到不同连接状态、在 connectable 状态下挂载聊天入口 | 部分：聊天入口依赖的 `agentName` / `runtimeUrl` 已进入统一配置中心；左侧列表仍不持久化 | 部分：右侧聊天入口会在 `ready` / `degraded` 时接到 runtime；左侧列表未接真实后端 | 当前真正接近主路径的是右侧聊天入口，不是整个三栏都已产品化 | 继续补真实消息流、真实会话数据与更完整聊天体验 |

| `files` | 前端静态文案、条目列表和概念分区 | 低：主要是进入页面和浏览内容 | 否 | 否 | 当前主要表达未来文件工作区信息架构 | 后续可接文件索引、附件管理、资料导入和知识检索数据 |
| `developer` | 前端静态文案、条目列表和概念模块 | 低：主要是进入页面和浏览内容 | 否 | 否 | 当前主要表达未来开发工作区结构 | 后续可接任务状态、构建结果、日志、验证记录和调试面板 |

## 必要说明

### 1. 为什么 `assistant` 写成“部分持久化、部分接 runtime”

因为当前只有 `assistant` 右侧聊天入口这条主路径，会真正消费：

- 配置中心中的 `agentName`
- 配置中心中的开发态 `runtimeUrl`
- 主进程提供的 hosted runtime 快照

但这并不代表整个 `assistant` 工作区都已经接通：

- 左侧助手类型仍是静态列表
- 中间话题列表仍是静态列表
- 新建话题按钮仍为占位

所以它既不能写成“完全占位”，也不能写成“完整联调页面”。

### 2. 为什么 `settings` 看起来最完整，却仍不能写成“全部已生效”

因为“页面很完整”和“配置已经正式进入系统主链路”是两回事。

当前设置页里，比较明确已经进入正式链路的只有：

- 主题
- 默认 Agent 名称
- 开发态运行时覆盖地址

另外：

- `model` 已进入配置中心 schema 和主进程投影逻辑
- 但当前没有正式 UI 入口

所以这页现在最准确的定位是：

- **当前事实**：设置页结构与交互已经较完整
- **正式接入**：只有少量关键字段已经进入统一配置中心闭环
- **仍是占位**：其余大多数字段仍只是本地交互或未来方向表达

### 3. 为什么后三个工作区都写成“低交互、未持久化、未接后端”

因为当前 `capabilities`、`files`、`developer` 的共同特征很明确：

- 有独立入口
- 有独立页面结构
- 有标题、说明、亮点和条目
- 但内容主要来自前端静态数据
- 没有真实数据回流
- 没有业务闭环

所以它们现在更像未来页面的结构雏形。

## 当前已实现 / 未实现说明

### 当前已实现

- 五个工作区都已有独立入口
- `assistant` 和 `settings` 的页面完成度明显高于其余三个工作区
- `assistant` 右侧已有最小聊天主路径
- `settings` 已经开始承载统一配置中心正式入口
- `capabilities`、`files`、`developer` 已有清晰的页面主题和结构雏形

### 当前未实现

- `assistant` 左侧列表和真实会话数据接通
- `settings` 中大多数字段的统一持久化和正式生效
- `settings` 中 `model` 的正式 UI 入口
- `capabilities` 的真实工具状态和调用结果
- `files` 的真实文件数据、导入链路和知识索引
- `developer` 的真实日志、任务与调试面板
