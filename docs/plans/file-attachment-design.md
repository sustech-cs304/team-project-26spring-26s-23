# 文件附件设计

## 设计核心

当前的 Agent 系统需要设计一个识别粘贴文件/图片到对话框，以附加文件的功能。具体期望功能如下：

1. 纯文字的粘贴，如纯文本、代码等，直接作为纯文本输入到对话框中，不进行任何特殊处理，这也是默认行为。
2. 当用户粘贴文件，或者纯数据（如图片数据）时，系统能够识别出这是一个文件，并将其作为附件进行处理，而不是直接将内容粘贴到对话框中。
3. 用户可能会同时粘贴多个文件或者数据，系统应该能够识别并处理这些情况。
4. 用户可能尝试通过拖拽文件到对话框中，这种情况也应该被识别并处理。
5. 对于无法明确文件类型的纯数据，系统应该弹出一个很轻量的浮动提示该数据类型不支持，并且不进行任何处理。

对于文件和纯数据，具体来说，系统需要能够区分以下几种情况：

1. 该文件是一个明确的、带有路径的文件，例如用户从文件管理器中复制了一个文件，这个时候系统应该提取出这个文件的路径作为附件进行处理
2. 该文件是一个纯数据，例如用户从某个应用中复制了一个图片或者其他类型的纯数据，这个时候系统应该将这个数据根据数据类型立刻写入文件到临时目录，然后将这个文件路径作为附件进行处理。

当用户点击发送按钮时，应该将当前附加的所有文件路径作为输入发送给 Agent，具体来说，发送给 Agent 的 Prompt 会自动在末尾附加一段对话，提醒 Agent 当前有文件附件，格式应该是：

```plaintext
User attached files:
- file1_path
- file2_path
- ...
Please process these files accordingly, for example, use `read_file` tool to read the content of these files.
```

也就是说，最终系统得到的一定是一个路径列表。这样设计的理由是，Agent 执行时会自动调用读取文件工具，因此应该将文件路径列表作为输入，而不是直接将文件上传到服务提供商。这样可以保持系统的一致性和可扩展性，同时也方便后续的文件处理和管理。

## UI

在 UI 上，应该进行如下设计：

1. 首先，如果用户附加了任何文件，应该向左渐入一个位于发送按钮左侧的浮动图标，这个图标代表了附加文件，并且用一个浮动的小文字角标代表“当前附加了多少个文件”，这个图标应该在用户附加了文件后持续显示，直到用户发送消息或者取消附加文件。
2. 当用户点击这个图标时，应该弹出一个浮层，这个浮层位于附加文件图标上方，浮层中应该列出当前附加的所有文件，每个文件应该显示文件名和一个小的删除图标，用户可以点击删除图标来取消附加该文件。
3. 浮层中特别处理：
   - 如果附加的文件是一个图片类型：直接显示一个略缩图+文件名，而非单纯的文件名，并且用户点击略缩图或者文件名都可以预览这个图片，预览方式可以是一个模态框，模态框中显示这个图片的完整内容。
   - 如果附加的文件是文本类型：直接显示文件名，用户点击文件名可以预览这个文本内容，预览方式同样是一个模态框（可滚动），模态框中显示这个文本的完整内容。
   - 其它类型：直接显示文件名，但是用户点击文件名不进行预览，静默忽略。
4. 当用户点击发送按钮时，应该将当前附加的所有文件路径作为输入发送给 Agent，并且在发送完成后清空附加文件列表，隐藏附加文件图标和浮层。
5. 当用户取消附加文件时，应该立即更新附加文件列表和图标显示，如果没有任何附加文件了，应该隐藏附加文件图标。
